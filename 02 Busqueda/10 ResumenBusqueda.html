Comparacin de algoritmos de bsqueda

Los diferentes algoritmos de bsqueda tienen una fundamentacin bsica que consiste en la construccin de rboles de bsqueda para encontrar soluciones a partir de un estado inicial. Dependiendo de las diferentes estrategias que se utilicen para explorar el rbol de bsqueda, hablamos de diferentes algoritmos.

Cada algoritmo tiene sus complejidades y, en general, pueden usarse para diferentes tareas. En particular los algoritmos desinformados sern usados cuando no contamos con informacin, ni ninguna heurstica que nos dirija. En otro caso, preferiremos algoritmos informados, pues estos tendern a ser ptimos.

Recordemos que tomaremos en consideracin la siguiente:

  • Completo: es decir, si el algoritmo se garantiza que siempre encontrar una solucin y por tanto ha de terminar.
  • ptimo: que el algoritmo encuentre la solucin ptima siempre que encuentre la solucin.
  • Complejidad: el nmero de nodos expandidos durante el proceso de bsqueda.

Creacin del problema

Para poder comparar de manera ms efectiva los algoritmos, definiremos un problema en el que conocemos la heurstica ptima; se trata del mundo del laberinto, pero sin ningn obstculo, por lo que slo buscaremos llegar del punto A al punto B. En este caso, ya que estamos en un mundo discreto, cuadriculado, donde el agente no puede hacer movimientos en diagonal, la heurstica ptima es la distancia Manhattan.

Heurstica

Definimos entonces una heurstica en base a la distancia, en donde podamos variar los valores (euclideana, Manhattan) a partir del parmetro $p$:

$$h(n) = \Big(|x_n - x_{goal}|^p+|y_n-y_{goal}|^p \Big)^{1/p}$$

Si $p=1$ tenemos la distancia Manhattan, y si $p=2$ tenemos la euclideana.

def heuristic(state, goal, p=2):
    """Funcin de heurstica de distancia Minkowsky"""
    x,y = state
    x_goal, y_goal = goal
    distance = (abs(x - x_goal)**p + abs(y - y_goal)**p)**(1/p)
    
    return distance

class PriorityQueue(object):
    """Clase de una cola de prioridad en los problemas de bsqueda"""
    def __init__(self, k=None):
        self.queue = []
        self.size = k
  
    def __str__(self):
        return ' '.join([str(q) for q in self.queue])
  
    def isEmpty(self):
        return self.queue == []
  
    def push(self, element):
        if self.size == None:
            self.queue.append(element)
        else:
            if len(self.queue) >= self.size:
                for item in self.queue:
                    if item.f > element.f:
                        self.queue.remove(item)
                        self.queue.append(element)
                    else:
                        pass
            else:
                self.queue.append(element)        
  
    def pop(self):
        min_element = np.argmin([element.f for element in self.queue])
        item = self.queue[min_element]
        del self.queue[min_element]
    
        return item
    
    def top(self):
        min_element = np.argmin([element.f for element in self.queue])
        item = self.queue[min_element]
        
        return item
    
class Node(object):
    """Clase para crear nodos con sus atributos"""
    def __init__(self):
        self.state = None
        self.parent = None 
        self.action = None
        self.path_cost = 0
        self.depth = 0
        self.f = 0
        
    def __str__(self):
        return str(self.state)
    
def expand(problem, node, method='Astar',p=1):
    """Funcin para expandir los nodos dado el problema"""
    s = node.state
    for action in problem.actions(s):
        new_s = problem.result(s, action)
        
        new_node,new_node.state,new_node.parent,new_node.action = Node(),new_s,node,action
        new_node.depth = node.depth + 1
        new_node.path_cost = heuristic(new_s,problem.initial,p=2)
        if method == 'Astar':
            new_node.f = new_node.path_cost + heuristic(new_s,problem.goal,p=p)
        elif method == 'Dijkstra':
            new_node.f = new_node.path_cost
        elif method == 'Breadth':
            new_node.f = new_node.depth
        elif method == 'Depth':
            new_node.f = -new_node.depth
        else:
            print("No es algoritmo vlido")
            
        yield new_node

def canvas2rgb_array(canvas):
    """Adapted from: https://stackoverflow.com/a/21940031/959926"""
    canvas.draw()

Algoritmos de bsqueda

Definimos una funcin general que tomar en cuenta los siguientes parmetros:

  1. method: Indica qu algoritmo se usar, consideramos los siguientes:
    1. Bsqueda en Profundidad
    2. Bsqueda en Amplitud
    3. Algoritmo de Dijkstra
    4. $A^*$
    5. Beam Search
  2. p: Controla la heurstica, si $p=1$ se usa Manhattan, si $p=2$ se usa euclideana. Se pueden probar otros valores, considerando que den valores reales. Slo aplica en $A^*$.
  3. beam: si en None No se usar Beam Search, si se escoge un $k>0$ se usar beam search. Se puede combinar con distintos algoritmos.

def search(problem, method, p=2, beam=None):
    """Algoritmos de bsqueda"""
    nodes = []
    node = Node()
    node.state = problem.initial
    frontier = PriorityQueue(k=beam)
    frontier.push(node)
    reached = {str(problem.initial):node}
 
    while frontier.isEmpty() == False:
        node = frontier.pop()
        nodes.append(node)
        if node.state == problem.goal:
            return nodes
        
        for child in expand(problem, node, method=method, p=p):
            state = child.state
            if str(state) not in reached.keys():
                reached[str(state)] = child
                frontier.push(child)

    print("No se ha logrado llegar a un estado final.")
    return nodes

Algoritmo primero en profundidad

El algoritmo de primero en profundidad es quiz de los menos recomendados pues no se asegura que sea completo. En general, este algoritmo persiste en una accin, la ms profunda, hasta que no puede realizarla ms y realiza otra, pero entonces sigue persistiendo en la accin de nuevo, repitiendo este proceso hasta encontrar solucin. En este caso, consideramos la funcin de prioridad como:

$$f(n) = -depth(n)$$

Algoritmos informados

Algoritmo $A^*$

El algoritmo $A^*$ es el algoritmo de bsqueda heurstica por excelencia. Este se basa en una funcin heurstica $h:V\to \mathbb{R}$ que nos informa qu tan lejos estamos de la meta. Su funcin est dada por:

$$f(n) = g(n) + h(n)$$

Donde $g(n)$ es el costo y $h(n)$ es la heurstica. Es un algoritmo completo, sin embargo, es ptimo si la heurstica es admisible; esto es, si $h$ cumple que para toda $n$:

$$h(n) \leq h^*(n)$$

Donde $h^*(n)$ es la heurstica ptima. Adems, su complejidad depender tambin de la heurstica. Es decir, tenemos que:

S Depende de $h$ Depende de $h$

Conclusiones

En general, los algoritmos revisados tienen ciertas ventajas y desventajas. Pero el que muestra mejor desempeo es el algoritmo de $A^*$ principalmente cuando su heurstica es ptima o cercana a la ptima. Sin embargo, no siempre se puede encontrar una heurstica de este tipo, sobre todo cuando nuestros entornos son ms complejos (como parcialmente observables, dinmicos, etc.); en estos casos, segn la informacin que se cuente se puede usar un algoritmo de bsqueda desinformada.

Los algoritmos como Beam Search tambin se utilizan bastantes para diferentes tareas, pues permiten acostar la bsqueda a los mejores resultados. Esto hace ms eficiente el procesamiento sobre todo cuando se tiene un nmero grande de datos.
