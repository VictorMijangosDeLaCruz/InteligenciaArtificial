Modelos Ocultos de Márkov¶

Ejemplo de Etiquetado en Texto¶

El algoritmo de Viterbi es un algoritmo dinmico ampliamente usado para estimar probabilidades de forma eficiente dentro de los Modelos Ocultos de Mrkov. La idea general del algoritmo de Viterbi es realizar una bsqueda por caminos que sean altamente probables desde el principio, descartando aquellos caminos que tengan menos probabilidad. De esta forma, el algoritmo de Viterbi recorrer nicamente los caminos que tengan la mayor probabilidad para cada una de los posibles valores de las variables de emisiones $Y_t$, con $t=1,2,...,T$. Si bien esto no garantiza que se haya encontrado la probabilidad mxima global, si nos puede llevar hacia mximos locales que nos ayuden a resolver el problema.

En este notebook presentamos una implementacin del algoritmo de Viterbi para realizar etiquetado de partes de la oracin en texto. Esta es una tarea comn en lingstica computacional que consiste en asignar a las palabras en un texto su categora gramaticas; es decir, si son verbos, adjetivos, sustantivos, etctera.

Para esto, debemos obtener, a partir del texto, los valores de las observaciones y las emisiones; para hacer esto, definimos dos funciones que nos permitirn crear estos elementos. Como estamos trabajando con texto de lenguaje natural, nuestras observaciones y emisiones consistirn en vocabularios.

Estimación del Modelo Oculto de Márkov¶

Recurdese que un Modelo Oculto de Mrkov est definido como $HMM = (\mathcal{Y}, \mathcal{X}, A, \Pi, B)$ donde cada elemento es:

  1. $\mathcal{Y} = \{y_1, y_2,...,y_n\}$: corresponde a un conjunto finito de smbolos de emisiones; es decir, de valores que las variables $Y_t$ pueden tomar. En este caso, se asume que las $Y_t$ son idticamente distribuidas, por lo que sin importar el tiempo $t$ en el que se encuentren pueden tomar valores dentro del mismo conjunto $\mathcal{Y}$. En este caso, $\mathcal{Y}$ corresponde a las etiquetas gramaticales.
  2. $\mathcal{X} = \{x_1, x_2,...,x_n\}$ son los valores que las variables $X_t$ pueden tomar. De igual forma, asumimos que para cada tiempo $t$, las variables $X_t$ son idnticamente distribuidas y slo toman valores de $\mathcal{X}$. En este caso, corresponde a las palabras del lenguaje espaol.
  3. $A = (a_{i,j}) = p(y_j | y_i)$: corresponde a las probabilidades de transicin sobre las emisiones $\mathcal{Y}$. En este caso, se representan las probabilidades $p(Y_{t+1} = y_j | Y_t = y_i)$, pero asumimos que el proceso es homogneo, esto es, que la probabilidad de transicin es siempre la mismta, sin importar el tiempo en que no encontremos, por lo que omitimos las variables $Y_t$.
  4. $\Pi = (\pi_i) = p(y_i)$: es la probabilidad de que el smbolo $y_i$ sea inicial; es decir, de que inicie el proceso $P(Y_0 = y_i)$, de nuevo, omitimos la variable $Y_0$.
  5. $B = (b_{i,j}) = p(x_j|y_i)$: es la matriz de probabilidades de observaciones, esto es, la probabilidad de que $y_i$ genere a $x_j$ o $p(X_t = x_j | Y_t = y_i)$. De nuevo, ya que el proceso es homogneo, podemos omitir las vairables que indican el tiempo.

Para obtener la matriz $B$ de probabilidades de emisin, requerimos saber que observaciones son generadas por qu emisiones. Para esto obtendremos pares de la forma $(y_i,x_j)$ con $x_j \in \mathcal{X}$ y $y_i \in \mathcal{Y})$. A partir de esto, simplemente haremos un conteo de las veces que una etiqueta y una palabra aparecen juntas.

Ahora obtendremos las estimaciones de probabilidad para las transiciones y el inicial sobre los smbolos en $\mathcal{Y}$. Para que las probabilidades sean adecuadas, introduciremos dos nuevos smbolos:

Este smbolo es necesaria para garantizar esto cuando realizamos las probabilidades por conteo, ya que de otra forma, las palabras que terminan una oracin estaran incompletas.

Obtención de los parámetros del modelo¶

Una vez obtenido los vocabularios, as como sus indices y los pares de observaciones y sus emisiones, adems de los pares que determinan las transiciones de las emisiones, podemos determinar el modelo; es decir, estimaremos las probabilidades que llenarn las marices $A$, $\Pi$ y $B$.

En este caso, generaremos un vector de iniciales, y dos matrices, una para las transiciones y otras para las emisiones. En inicio, estas matrices comenzarn con 0's y se irn llenando con las probabilidades correspondientes.

En primer lugar, comenzaremos obteniendo las probabilidades de $p(Y_0)$ que corresponden a lo que en el modelo se conocen como probabilidades iniciales. Para esto, calcularemos $p(Y_0=y_i)$ o simplemente $p(y_i)$ apoyndonos del smbolo BOS que hemos introducido:

$$p(y_i) = \frac{\#(BOS, y_i) + 1}{\# BOS + N}$$

En este caso, $\#(BOS, y_i)$ responde a las veces en que el smbolo de emisin $y_i$ aparece despus del smbolo BOS (esto corresponde a las veces que $y_i$ inicia una secuencia, esto sobre $\# BOS$ que es el total de veces que se inicia una secuencia.

Es importante notar que aqu hemos sumado un 1 al numerador y un $N$ al denominador. Esto es un suavizamiento Laplaciano de la probabilidad y se realiza para evitar que haya probabilidades 0. Las probabilidades 0 son problemticas, pues cuando se hace el producto de las probabilidades un 0 puede llevar la probabilidad conjunta total a 0, a pesar de que otros valores sean altos. Adems, aqu $N$ es el total de entradas del vector, que corresponde al nmero de smbolos. Esto garantiza que la probabilidad suavizada siga siendo una medida de probabilidad, pues es fcil observar que:

$$\sum_{i=1}^N p(y_i) = \sum_{i=1}^n \Big(\frac{\#(BOS, y_i) + 1}{\# BOS + N}\Big) = \frac{\sum_{i=1}^n \#(BOS, y_i) + 1}{\# BOS + N} = \frac{\# BOS + N}{\# BOS + N} = 1$$

Para la matriz de transiciones hacemos algo similar, contamos el nmero de veces que el smbolo $y_i$ es seguido por el smbolo $y_j$ y lo dividimos entre el total de veces que atestiguamos el smbolo $y_i$. Asimismo, usamos el suavizamiento Laplaciano para eliminar las posibles probabilidades 0.

$$p(y_j|y_i) = \frac{\#(y_i y_j)+1}{\#(y_i) + N}$$

Al final obtenemos tanto el vector de iniciales con entradas:

$$\Pi_i = p(y_i)$$

Y as mismo obtenemos la matriz de transiciones con las entradas:

$$A_{i,j} = p(y_j|y_i)$$

Cálculo bruto de cadena de emisiones¶

Podemos preguntarnos cmo obtener la mejor secuencia de emisiones dado una secuencia de observaciones. En este caso, esto implica estimar la probabilidad:

$$p(y^{(0)},y^{(1)},...,y^{(T)}| x^{(0)}, x^{(1)},...,x^{(T)})$$

Donde $x^{(0)}, x^{(1)},...,x^{(T)}$ es la cadena de observaciones (o evidencia) de entrada, que son conocidas. Por tanto, lo que buscamos son los valores $y^{(1)},...,y^{(T)}$ que maximicen esta probabilidad. Podemos observar que:

$$\arg\max p(y^{(0)},y^{(1)},...,y^{(T)}| x^{(0)}, x^{(1)},...,x^{(T)}) = \arg\max p(y^{(0)},y^{(1)},...,y^{(T)}, x^{(0)}, x^{(1)},...,x^{(T)})$$

Cuando el argumento que maximiza se computa sobre la secuencia $y^{(0)}, y^{(1)},...,y^{(T)}$. Esto nos remite a la estimacin por la red bayesiana. Sabemos que la estructura de esta red es como sigue:

En este caso, las variables $Y_t$ tiene como tabla de proabilidades condicionales a la matriz de transicin (i para $Y_0$ al vector de iniciales), mientras que las observaciones, al depender slo de la $Y_t$ correspondiente tienen como su tabla de probabilidades a la matriz de observaciones. Entonces, al ser una red bayesiana con la estructura previamente definida, sabemos que:

$$p(y^{(0)}, y^{(1)},...,y^{(T)}, x^{(0)}, x^{(1)},...,x^{(T)}) = p(y^{(0)})\prod_{t=1}^{T} p(x^{(t)}|y^{t})p(y^{(t)}|y^{(t-1)})$$

Sin embargo, para obtener la secuencia que maximiza esta cadena, tenemos que obtener todas las posibles combinaciones de $N$ smbolos de emisiones en $T+1$ elementos de la secuencia. Es decir, tenemos que realizar $N^{T+1}$ clculos, una complejidad exponencial. Podemos ver qu pasa si intentamos hacer esto.

Algoritmo de Viterbi¶

El algoritmo de Viterbi es un lagoritmo dinmico que obtiene las probabilidades de caminos que maximizan cada una de sus transiciones.

El algoritmo nos permitir obtener las emisiones que maximicen la probabilidad conjunta con las observaciones. Podemos observar que las posibles combinaciones de observaciones son $N^{T+1}$ donde $N$ es el nmero de smbolos de emisin y $T$ la longitud de la observacin. La complejidad del algoritmo anterior es entonces $O(N^T)$, pero el algoritmo de Viterbi nos permitir reducir esta complejidad significativamente.

Se puede pensar como un algoritmo de bsqueda en donde en cada transicin se elijen slo los caminos previos que maximizan la probabilidad de llegar al estado actual. En primer lugar, observemos que cuando estimamos la probabilidad de una secuencia de emisiones $y^{(0)}, y^{(1)},...,y^{(T)}$, podemos pensar a $y^{(0)}$ como un estado inicial; sin embargo, hay $N$ posibles estados iniciales, correspondientes a los valores que puede tomar esta vairable; es decir, podemos iniciar desde cualquier variable $y_i$ con probabilidad $\pi$. Transitar hacia el siguiente valor $y^{(1)}$ se puede hacer desde cualquiera de los $N$ valor iniciales y puede llegar a cualquiera de los $N$ valores actuales, por lo que tenemos $N^2$ posibles caminos con probabilidad $p(y_j|y_i)$. Esto mismo pasa para cada otro estado, por lo que en el tiempo $t$ tendremos que considera $N^t$ caminos posibles. Adems debemos considerar las observaciones.

El espacio de bsqueda se puede grficar en lo que se conoce como diagrama de Trellis. En nuestro caso particular, el diagrama de Trellis para la entrada dada se meustra en la siguiente figura.

Implementación del Algoritmo de Viterbi¶

Lo que busca el algoritmo de Viterbo es reducir el espacio de bsqueda a slo aquellos caminos que sean los ms probables en cada tiempo $t$. Para esto, se basa en dos pasos generales:

  1. Forward (avance): En esta parte busca calcular las probabilidades y los caminos que tengan la mayor probabilidad para cada tiempo $t$ olvidando aquellos caminos que en este tiempo no maximicen la probabilidad. Al final se quedar con slo $N$ caminos posibles.
  2. Backward (retroceso): De los $N$ caminos posibles elige al que tenga la mayor probabilidad total y recupera las emisiones correspondientes a este camino.

La salida del algoritmo ser la secuencia de emisiones que mazimiza la probabilidad de los $N$ caminos restantes durante el paso forward. Como sealamos, no se puede garantizar que el algoritmo de Viterbi llegue al mximo global, pero si garantiza un mximo local, y en la mayora de los casos funciona adecuadamente.

Forward¶

El paso forward define una variable $\delta_j(t)$, donde $j$ corresponde acada una de las $N$ valores que pueden tomar nuestras variables $Y_t$ en el tiempo $t$. Asimismo, define variables categricas $\phi_j(t)$ que lo que hacen es guardar los caminos definidos por las probabilidades de las variables $\delta_j(t)$. El paso forward consiste en:

  • Inicializacin: Inicializa los valores $\delta$ para cada $j$ correspondiente a los valores de emisin como: $$\delta_j(0) = p(x^{(0)}|y_j)p(y_j)$$ En trminos de matrices $$\delta(0) = B_{x^{(0)}} \odot \Pi$$
  • Induccin: Obtiene las probabilidades en cada tiempo $t$ quedndose slo con aquellas que maximizan los valores, de igual forma lo hace para cada valor $j$: $$\delta_j(t+1) = \max_i p(x^{(t+1)}|y_j)p(y_j|y_i)\delta_i(t)$$ En trminos de vectores y matrices: $$\delta(t+1) = \max_i A \odot \Big(B_{w^{(t+1)}} \otimes \delta(t) \Big)$$ Las variables $\phi$ se calculan de forma similar pero usando el $\arg\max$.

Podemos observar que lo que obtiene son las probabilidades en cada tiempo $t$ para $\delta$ de la siguiente forma:

$$\delta_j(t) = \max_{i_1,...,i_{t-1}} p(y_{i_1}^{(1)},..., y_{i_{t-1}}^{(t-1)}, x^{(1)},...,x^{(t)}, y^{(t)}_j)$$

Es decir, las variables $\delta$ guardan las probabilidades conjuntas para cada tiempo $t$, por lo que en el ltimo paso $T$, tendremos $N$ variables $\delta_j$ cada una con una probabilidad conjunta. El caso de $\phi$ es similar:

$$\phi_j(t) = \arg\max_{i_1,...,i_{t-1}} p(y_{i_1}^{(1)},..., y_{i_{t-1}}^{(t-1)}, x^{(1)},...,x^{(t)}, y^{(t)}_j)$$

En este caso, lo que las variables $\phi_j$ guardan son los valores de los caminos que maximizan las probabilidades.

Backward¶

El paso backward se basar en la salida del paso forward, pues lo que har es recuperar el camino, de los caminos obtenidos en el forward, que maximice la probabilidad total. De esta forma, lo nico que hace es encontrar el camino con mayor probabilidad y recuperar los valores que le corresponden. Este oalgoritmo entonces se basar en:

  • Inicializacin: Encuentra el camino que tiene la mayor probabilidad total y recupera el ltimo smbolo de este camino como: $$\hat{y}^{(T)} = \arg\max_i \delta_i(T)$$
  • Induccin: Recupera el valor previo en el camino elegido en la inicializacin de la forma: $$\hat{y}^{(t)} = \phi_{\hat{y}^{(t+1)}}(t+1)$$

Como podemos observar, la inicializacin busca la probabilidad ms alta en las variables $\delta_i(T)$ y la induccin recupera el camino con las variables $\phi$. El camino lo recupera de manera inversa, pues empieza del ltimo valor y de all recorre hacia atrs (de all el nombre backward) hasta el primer smbolo. Por lo que la secuencia debe ser invertida para obtener la secuencia real.

def Viterbi(obs_text, HMM):
    """Algoritmo de Viterbi"""
    #Obtiene el HMM
    ems_voc, obs_voc, A, Pi, B = HMM
    #Forward
    d,phi = forward(obs_text, (ems_voc, obs_voc, A, Pi, B))
    #Backward
    tags_idx = backward(d,phi)
    
    #Recupera los tags de los ndices
    tags = {value:key for key, value in ems_voc.items()}
    output = [tags[yt] for yt in tags_idx]
    
    print( 'Input:\t{}\nOutput:\t{}'.format('\t'.join(obs_text), '\t'.join(output)) ) 
    
    return output
