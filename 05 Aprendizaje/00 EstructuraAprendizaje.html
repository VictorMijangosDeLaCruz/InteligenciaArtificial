Estructura general para Aprendizaje Automtico

El aprendizaje automtico o estadstico consiste en una serie de tcnicas que se basan en el principio general de aprendizaje: decimos que el sistema aprende si mejora su rendimiento en una tarea a partir de la experiencia. Esta definicin de aprendizaje incorpora 3 elementos bsicos:

  1. La tarea que responde al objetivo que el agente busca alcanzar o el problema que se quiere solucionar. Por ejemplo, la tarea puede ser la clasificacin de e-mails, el reconocimiento de imgenes, etc.
  2. El rendimiento que requiere de una medida cuantitativa para determinar qu tan bien resuelve la tarea nuestro sistema.
  3. La experiencia; es decir, aquel material que enriquecer los conocimientos (estados de creencia) de nuestro agente. En este caso la experiencia refiere a los datos con los que aprendemos.

Siempre que se aplique un algoritmo de aprendizaje automtico, es importante seguir una metodologa que nos permita determinar qu tan bien el modelo de aprendizaje generaliza. Esta metodologa consistir en los siguientes pasos:

  1. Determinar el conjunto de datos (experiencia) con el que se va a trabajar. Este conjunto de datos debe ser idealmente representativo del problema, sea este supervisado o no supervisado (o de otro tipo). El conjunto de datos debe separarse en varios conjuntos:
    1. Datos de entrenamiento: Corresponden a aquellos datos con los que se aprender el modelo. Estos datos corresponden a la experiencia propiamente dicha, pues son aquellos datos que nuestro agente de aprendizaje ver y a partir de los cuales aprender. Generalmente se toma un 70% del conjunto de datos original para el entrenamiento.
    2. Datos de validacin: Son datos que pueden servir para ajustar hiperparmetros. No siempre son necesarios.
    3. Datos de evaluacin: Son los datos con los que se determinar el rendimiento del agente de aprendizaje. Es importante que los datos de evaluacin no deben haber sido vistos por el agente previamente (durante el entrenamiento), pues de otra forma no podremos decir que el agente est aprendiendo (si no memorizando). Generalmente, se usa un 30% para los datos de evaluacin.
  2. Seleccin de los modelos: Despus de que tenemos el problema a tratar, podemos pasar a hacer una seleccin de un modelo para que resuelva este problema. En primer lugar debemos determinar qu tipo de problemas estamos enfrentado; podemos pensar en los siguientes tipos de problemas:
    1. Supervisados: Cuando tenemos un indicador de la clase que deseamos de salida: $\mathcal{S} = \{x,y) : x \in \mathbb{R}^d. y \in Y\}$
    2. No-supervisados: Cuando slo contamos con los datos del problema, pero no con una supervisin; es decir, el conjunto es de la forma $\mathcal{U} = \{x \in \mathbb{R}\}$
    3. Podemos hablar de otros tipos de aprendizaje; como el aprendizaje por refuerzo, donde el agente busca resolver una tarea con una funcin de recompensa; aprendizaje auto-supervisado donde el agente genera su propia supervisin; o aprendizaje semi-supervisado.
  3. Entrenamiento del modelo: Una vez que se cuenta con los datos y se ha seleccionado un modelo adecuado para el problema, se proceder a entrenar el modelo. Este proceso es lo que se puede llamar aprendizaje propiamente dicho, pues es aqu donde el agente, a partir de la observacin de los datos, aprende a resolver el problema. En este paso se podrn seleccionar hiperparmetros (valores del modelo que son elegidos por el programador) por lo que se puede usar una parte del dataset original (generalmente el 10%) para poder determinar esos valores; a esto corresponde el paso de validacin.
  4. Evaluacin del modelo: Este ltimo paso es de suma importancia, pues determina la capacidad que el agente de aprendizaje tiene para generalizar. La evaluacin del modelo consiste en determinar una medida numrica que nos indique qu tan bien el agente resuelve el problema para datos que no ha visto (de los que no ha aprendido). Segn el tipo de problema se pueden utilizar diferentes mtricas de evaluacin.

2. Seleccin de algoritmos

Ya que tenemos un problema que cuenta con un conjunto de datos supervisados (en donde teneos muestras de las salidas que deseamos obtener), se propone el uso de tres algoritmos supervisados. Cada uno de estos algoritmos pertenece a una familia especfica en la que se enfoca en la clasificacin supervisada:

  1. Algoritmos paramtricos: Los algoritmos paramtricos dependen de un grupo de parmetros que describen una distribucin probabilstica. En este caso, tomamos dos algoritmos paramtricos:

    1.1. Bayes Nave: El algoritmo de Bayes Nave se basa en una red bayesiana simple. Se trata de un modelo grfico dirigido y, por tanto, generativo cuyo modelo se describe como: $$\hat{y} = \arg\max_y p(y,x_1,x_2,...,x_d; \theta) = \arg\max_y \prod_{i=1}^d p(x_i|y)p(y)$$ Es decir, estima una distribucin conjunta que se factoriza asumiendo que los rasgos $x_i$ son generados por una clase $y$. En este caso, se estima un conjunto de parmetros que responde a los factores $p(x_i|y)$ y $p(y)$; por tanto, se trata de un algoritmo paramtrico.

    1.2. Regresin logstica: La regresin logstica es un modelo grfico no dirigido y, por tanto, discriminativo en donde la clase se puede obtener como: $$\hat{y} = \arg\max_y p(y|x_1,...,x_d; \theta) = \arg\max_y \frac{1}{1+exp\{-\sum_{i=1}^d \theta_i x_i + \theta_0\}}$$. En este caso se estiman los parmetros $\theta_i$, $i=0,1,...,d$ que definen los factores de la distribucin. Por tanto, es tambin un algoritmo paramtrico.

  2. Algoritmo no-paramtrico: Para contrastar con los algoritmos paramtricos, tomamos un algoritmo no paramtrico: rboles de decisin, se trata de un algoritmo que construye un rbol binario donde cada nodo representa a un rasgo que se ramifica en dos casos: el dato contiene al rasgo o no lo contiene, o bien si son datos numricos si el rasgo es mayor o no a un umbral. Para obtener el rbol de decisin minimizamos la impureza de Gini definida como: $$Gini = 1 - \sum_{i=1}^c p_i^2$$

4. Evaluacin de resultados

La evaluacin consiste en determinar qu tan bien generaliza el modelo en datos nuevos. Podemos distinguir diferentes tipos de evaluacin segn el tipo de problema; en este caso, usaremos una evaluacin supervisada: es decir, ya que contamos con las clases o salidas que esperamos del algoritmo, comparamos estos valores con los valores arrojados por el modelo. En este caso, usamos 3 de las mtricas ms populares para hacer evaluacin supervisada (basadas en matriz de confusin). Para evaluar los resultados utilizamos, entonces, las mtricas usuales:

  • Precisin: $$Prec = \frac{TP}{TP+TN}$$
  • Exhaustividad: $$Rec = \frac{TP}{TP+FN}$$
  • Mtrica F1: $$F_1 = \frac{2 Prec*Rec}{Prec + Rec}$$

Donde $TP$ son los verdaderos positivos y $TN$ los verdaderos negativos. Adems, reportamos los promedios macro y pesado.

  • Macro-Promedio: $\frac{1}{|C|}\sum_{c\in C} m(y, \hat{y})$
  • Promedio ponderado: $\sum_{c \in C} p(c) m(y_c, \hat{y})$

Donde $m$ es una mtrica (precisin, recall, $F_1$) y $C$ es el conjunto de clases y $p(c)$ es la probabilidad de la clase $c$.

En conclusin, para el problema planteado que consiste en la deteccin de cncer a partir de la descripcin de los tumores, el algoritmo que tuvo mejor desempeo fue la regresin logstica. Esto se puede deber a que las descripciones son numricas (radio, permetro, textura) y la regresin logstica maneja bien este tipo de datos, pues pondera los valores numricos al multiplicarlos por pesos. Mientras que los rboles de decisin, por ejemplo, manejan mejor datos categricos de respuesta s o no. Asimismo, el modelo de Bayes Nave controla los datos por medio de la media y la varianza. Parece entonces que ponderar estos datos multiplicndolos por un valor real tienen mejores resultados que otros mtodos.
