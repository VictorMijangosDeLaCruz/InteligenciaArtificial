Q-learning en el ambiente del Frozen Lake¶

Alicia Muiz Jimnez

La clave para resolver problemas de aprendizaje por refuerzo (RL) es encontrar la poltica ptima ($\pi^{*}$) de la funcin de valor para el problema especfico. El Q-learning es un algoritmo del tipo modelo libre del RL basado en valor y es un policy off-learner, es decir, depende del ensayo y error para actualizar su experiencia y conocimiento del entorno, ya que tienen que aprender la dinmica del sistema a travs de la experiencia y aprende el valor de la poltica ptima independientemente del tipo de acciones del agente, as como que las actualizaciones de la funcin de valor se basan en la ecuacin de Bellman (Jang, Kim, Harerimana y Kim, 2019).

Su Q proviene de quality (cualidad en ingls), ya que el Q-learning representa qu tan til es una accin en ganar una recompensa a futuro (Shyalika, 2019). Es as que el Q-learning estima la funcin $Q(s,a)$ que es el valor esperado de hacer una accin 'a' en un estado 's' para estimar la poltica ptima y la ecuacin de actualizacin de los valores $Q(s,a)$ es la siguiente:

$$ Q(s, a) \leftarrow Q(s, a) + \alpha [ r(s, a) + max Q' (s',a') - Q(s, a) ] $$

Donde:

  • $r(s,a)$ = recompensa obetenida en ese estado dada una accin.
  • $$ = factor de descuento para el estado futuro (al que se transitar), su valor es fijo y puede tomar el rango de valores de [0,1].
  • $$ = parmetro de aprendizaje, no es fijo ya que al inicio comienza en 1 y con cada iteracin ir disminuyendo su valor hasta alcanzar el valor de 0 o un lmite inferior definido de antemano (ej. 0.01).
  • $Q(s, a)$ = valor Q del estado actual dada la accin a.
  • $max Q'(s', a')$ = mxima futura recompesa esperada del estado nuevo al que se transitar.

Como se puede apreciar en la ecuacin anterior, el Q-learning usa el mtodo de diferencias temporales para la actualizacin o estimacin de los valores Q, ya que slo toma en cuenta el estado actual y el siguiente estado para estimar los valores Q (Matiisen, s. f.). Por otro lado, cuando el Q-learning es ejecutado se crea lo que se llama una tabla-Q, que es una matriz donde las filas normalmente representan cada estado posible y las columnas las acciones posibles en cada estado (Aggarwal, s.f. ). Por lo tanto, cada entrada de la matriz es un par estado-accin. La tabla siempre se inicializa con todos los valores en 0 y se actualiza cada entrada con la ecuacin de actualizacin de los valores Q, de acuerdo con lo que el agente explora en cada iteracin. La tabla al final sirve como referencia para que el agente pueda seleccionar la mejor accin basada en los valores Q.

Finalmente, es importante recalcar que en el Q-learning el agente trabaja con una poltica epsilon-greedy, mas esta poltica no tiene valores fijos, sino que al inicio el algoritmo tiene un epsilon de 1 y con cada iteracin este valor va disminuyendo hasta terminar en 0 o en un lmite inferior como 0.01, si es que as se especifica de antemano (Matiisen, s.f.). Esto se debe a que como el algoritmo no tiene conocimiento de la dinmica del sistema al inicio, entonces necesita explorar mucho para conocer el entorno, pero conforme va conociendo ms del entorno puede explotar cada vez ms y no solo explorar.

Despus de la breve introduccin al Q-learning, ahora se proceder con un ejemplo prctico sobre dos agentes agentes que interactan en el entorno virtual "Frozen Lake" del gimnasio OpenAI, esto para entender mejor el Q-learning y las diferencias que se pueden observar al compararlo con un agente que tiene una poltica de eleccin de comportamiento de acciones aleatorias.

Explicación del ambiente¶

La documentacin de la paquetera gymnasium describe las siguientes caractersticas del ambiente Frozen Lake (Farama Foundation, 2023):

  • Estado de acciones (4):

    • 0 = Izquierda
    • 1 = Abajo
    • 2 = Derecha
    • 3 = Arriba

  • Espacio observacional (16): Se utilizar el ambiente de Frozen Lake con una cuadrcula de tamao 4x4, por lo que el total de observaciones posibles (estados) son 16, como se puede ver en la imagen de abajo.

  • Recompenzas:

    • +1 si el agente alcanza la meta.
    • 0 si el agente cae en algn hoyo en el hielo.
    • 0 si el agente pasa por estados donde hay hielo congelado.

Agente con acciones aleatorias¶

Creacin del ambiente virtual de Frozen Lake

Para implementar un agente de aprendizaje supervisado; determinamos, en primer lugar, un ambiente. Este se determina por una cuadrcula de $n\times n$. En la paquetera de Gymnasium, se usa la funcin make y le indicamos que queremos un ambiente de Frozen Lake. Este ambiente, simula un entorno en que una gente se mueve con las acciones antes indicadas para llegar a una meta; en el ambiente hay obstculos, que son hoyos donde el agente puede caer. Como resultado, se necesitan indicar los siguientes elementos:

  1. Se especifica la estructura del entorno con una cuadrcula de $n\times n$ (en este caso $4\times 4$) con el comando desc = ["SFFF", "FHFF", "FFFH", "FFFG"] (se pueden variar el orden de sus elementos), donde:

    • S = posicin inical del agente (start)
    • F = hielo (frozen)
    • H = hoyo
    • G = meta (goal)

  2. El comando is_slippery=True implica que el suelo est resbaloso y hace que el agente se mueva con una probalidad de 1/3 a la casilla deseada y con un 1/3 de probabilidad a las casillas perpendiculares a la accin deseada. Si es False el agente se mueve a la casilla deseada siempre.

  3. El comando render_mode="human" sirve para visualizar al entorno grficamente, si es igual a "ansi" entonces los valores de las acciones se guardan se guardan en un string que contiene una representacin de texto de estilo terminal para cada paso temporal, aunque slo se visualiza este string si es solicitado.

Funciones¶

Tasa de aprendizaje (learning_rate)

En Q-learning el agente en los primeros episodios aprende mucho porque el agente no tiene un modelo del entorno, pero conforme va conociendo el entorno su tasa de aprendizaje ir disminuyendo, es decir, la tasa de aprendizaje es dinmica (no es fija).

Si se anliza el siguiente cdigo se puede ver al inicio la tasa de aprendizaje comienza en 1 y con cada iteracin ir disminuyendo su valor hasta alcanzar el lmite de 0.01. Los argumentos que toma la funcin son:

  • n = Episodio en el que est el agente (se aprecia en partes del cdigo ms adelante cuando se usa la funcin).
  • min_rate = Lmite inferior que puede alcanzar la tasa de aprendizaje. Este lmite se pone porque se quiere que el agente siempre siga aprendiendo en cierta medida, aunque ya tenga un buen modelo del entorno.
In [10]:
def learning_rate(n : int , min_rate=0.01 ) -> float  :
    """Tasa de decaimiento del aprendizaje por cada episodio"""

    return max(min_rate, min(1.0, 1.0 - math.log10((n + 1) / 25)))

Tasa de exploracin (exploration_rate)

Hay que recordar que en Q-learning al princio se explora mucho porque el agente no tiene un modelo del entornoy debe explorar para aprender, pero con el paso de los episodios comenzar a explotar ms.

En el siguiente cdigo se puede ver que la tasa de exploracin se inicializa en 1 y va disminuyendo en cada episodio hasta alcanzar un lmite de 0.1. Los argumentos que toma la funcin son:

  • n = Episodio en el que est el agente (se aprecia en partes del cdigo ms adelante cuando se usa la funcin).
  • min_rate = Lmite inferior que puede alcanzar la tasa de exploracin. Este lmite se pone porque lo que se quiere es que el agente siga explorando de vez en cuando, aunque ya tenga un buen modelo del entorno.
In [11]:
def exploration_rate(n : int, min_rate= 0.1 ) -> float :
    """Tasa de decaimiento de exploracin en cada episodio"""
    #
    return max(min_rate, min(1, 1.0 - math.log10((n  + 1) / 25)))

Poltica epsilon-greedy

La poltica epsilon-greedy elige la opcin de mayor recompensa la mayora de las veces y con probabilidad elige aleatoriamente el resto de las veces (Francois-Lavent et al., 2018).

El cdigo a continuacin muestra que si un nmero aleatorio es menor que la tasa de exploracin, entonces se decide explorar, sino, se explota al elegir el argumento mximo de las acciones del Q-table (dado un estado especfico).

In [12]:
def policy_e_greedy(current_state : tuple):
    """Regla de decisin basada en una poltica epsilon-greedy, dado en el estado que se est"""

    # explorar (accin aleatoria)
    if np.random.random() < exploration_rate(e):
        action = env.action_space.sample()

    # explotar
    else:
        action = np.argmax(Q_table[current_state])

    return action

Actualizacin del valor Q

En Q-learning se emplea el mtodo de diferencias temporales para actualizar los valores Q (recordemos que un valor Q es la entrada de un par estado-accin de la tabla Q), porque para actualizar lo aprendido slo se toma en cuenta el estado actual y el siguiente.

El siguiente cdigo corresponde la ecuacin mencionada al inicio:

$$ Q(s, a) \leftarrow Q(s, a) + \alpha [ r(s, a) + max Q' (s',a') - Q(s, a) ] $$
In [69]:
def new_Q_value( reward : float , action, current_state, new_state : tuple , discount_factor=1 ) -> float:
    """
    Actualizacin del Q-value de un par estado-accn con el mtodo de diferencias temporales:

       reward = recompenza obtenida en el estado actual.
       action = accin realizada
       current_state = estado actual
       new_state = estado siguiente
       discount_factor = el peso que se le da al estado siguiente (entre ms cercano a 1, ms importa).

    """
    act_val = Q_table[current_state, action]                             #se obtiene el valor del estado actual
    fut_opt_val = np.max(Q_table[new_state])                             #se obtiene el valor del estado siguiente
    learned_value = reward + discount_factor * fut_opt_val-act_val       # r(s, a) + max Q' (s',a') - Q(s, a)
    new_val = (1-lr)*act_val + lr*(learned_value)

    return new_val

    #Estado inicial
    current_state = env.reset()[0]
    done = False
    num_episodio.append(e)
    score = 0

    actions = []
    while done==False:

        #Se selecciona la accin con la poltica epsilon greedy
        action = policy_e_greedy(current_state)
        actions.append(action_names[action])

        #Actualizacin de las variables en el ambiente
        #a partir de la accin tomada
        obs, reward, done, _, _ = env.step(action)

        new_state = obs
        score += reward
        lr = learning_rate(e)                         #se actualiza la tasa de arpendizaje

        #Se actualiza el valor Q
        Q_table[current_state, action] = new_Q_value(reward, action, current_state, new_state)

        #Se transita al nuevo estado
        current_state = new_state

    puntajes_ep.append(score)

    #impresin de resultados cada 100 episodios
    if e % 100 == 0:
        puntaje = puntajes_ep[e-1]

        print('Episode {}\n\tPuntaje:{}; Acciones:{}'.format(e, score, actions))
